Automated Ambient Light Control System

A system that brightens or dims LED according to the lighting conditions of its environment.

01 // Project Summary

Ambient light changes throughout the day causing inconsistent brightness in a workspace. In this project, I created an automated system that watches the room and dims or brightens the LED inversely to the room's light level. This system maintains a lighting setpoint so that it saves energy consumption for only when it is needed.

03 // Theory & Design

I used a voltage divider to convert the change in resistance from the photoresistor into a change in voltage that the microcontroller could understand. I programmed the system to map the analog input (0-1023) to a brightness level to create an inverse relationship.

Code for the System:

Programmed in C++

// Pin Definitions
const int sensorPin = A0;
const int ledPin1 = 9;
const int ledPin2 = 10;

// --- CALIBRATION ---
const int sensorDark = 67;    // Full Brightness (255) at this level
const int sensorCutoff = 800; // Full OFF (0) at this level
// -------------------

void setup() {
  Serial.begin(9600);
  pinMode(ledPin1, OUTPUT);
  pinMode(ledPin2, OUTPUT);
}

void loop() {
  int sensorValue = analogRead(sensorPin);

  // MAP FUNCTION
  int brightness = map(sensorValue, sensorDark, sensorCutoff, 255, 0);

  // CONSTRAIN
  brightness = constrain(brightness, 0, 255);

  analogWrite(ledPin1, brightness);
  analogWrite(ledPin2, brightness);

  Serial.print("Sensor: ");
  Serial.print(sensorValue);
  Serial.print(" | Brightness: ");
  Serial.println(brightness);
  
  delay(20);
}

05 // Challenges I Faced

  • Coding Inverse Logic: Programming the C++ map() function required multiple iterations to ensure the LEDs brightened smoothly as the room got darker. Flipping the relationship between the sensor input and the output without causing brightness spikes was a significant hurdle.
  • Signal Jittering: Ambient light can be unstable sometimes. Small fluctuations in the LDR readings initially caused the LEDs to "flicker." I solved this by implementing constraints and optimizing the loop delay to stabilize the feedback.
  • Sensor Non-Linearity: Photoresistors do not react to light linearly. Calibrating the sensorDark and sensorCutoff values required multiple testing using the Serial Monitor to match the sensor's physical response to the lighting in the room.
  • Hardware Safety: I had to ensure that the current draw across the LED network remained below the 20mA per-pin safety limit of the ATmega328P to prevent the component wearing out over time.

06 // Areas where this can be used

  • Smart Home Integration: Can be used to automatically adjust interior lighting to reduce energy waste by utilizing natural sunlight during the day.
  • Public Infrastructure: Energy-efficient street lights that adjust brightness based on dusk/dawn levels or weather conditions.
  • Automotive Safety: Auto-dimming dashboard backlighting and rearview mirrors to prevent driver eye strain and glare.
